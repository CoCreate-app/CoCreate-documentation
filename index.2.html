Introduction

CoCreateJS

A dependancy free, vanilla javascript Realtime CRUD and Collaboration API. Easily integrates in any HTML template using HTML5 data-attributtes. Converts a regular html website into a dynamic Realtime application. If you know basic html you can easily implement and use the CoCreateJS library/framework. If you have javascript knowledege, CoCreateJS is highly extendable Object oriented collaboration framework. A CoCreate api key is required to open a real time socket channels.

CoCreateCSS

A dependancy free, CSS and vanilla javascript library. Containing a set of easy to use natraul language helper classes, and a wide range of components to assist in developing modern UI/UX. We are excited to see what you build.

Micro Service Archeticture

CoCreateJS 90kb gizipped and CoCreateCSS 15kb gzipped were compiled from a large list of opensource projects. You can include and compile only the components you need, create custom components and contribute to a long list of existing componenets located on github

Getting Started

Go to CoCreate.app/register and retrieve your script which will include the necessary api and security key. And paste it at the end before the ending body tag of any page where you would like to fetch and save data in real time.

	
<html>
  <head>
    <link src="http://cdn.cocreate.app/CoCreate.min.css"></link>
  </head>
  
  <body>
  
	<script>
	  var config = {
   	  apiKey: 'test123456',
	  securityKey: 'test123456',
	  organization_Id: '5d0e4e6147fb95a3b8719f12'
	}
	</script>

    <script src="http://cdn.cocreate.app/CoCreate.min.js"></script>

  </body>
</html>

REALTIME HTML Elements

You will have the option to set realtime as true or false. If data-realtime set to false then the form will require a button to save(<button class="submitBtn">). And yes there is a method for redirecting using a btn and <a> tag which we will discuss at length further in this documentation.

We have 2 collections by default “modules” and “module_activity”. If the data-collection="" is left empty or if the attribute is not used it will default to “module_activity”.

If your logic requires a new collection just add it to the attribute (data-collection='custom collection name') and it will be created if it does not exist.

By adding attribute data-form_id=”a unique id” allows for a form creating a new record to be updateable in Realtime. It also differentiates the form from all other forms in the case that you may have multiple forms on the same page.

Attributes

  • data-realtime boolean optional

    Save data onChange. If attribute is empty or not used Default's to true.

  • data-collection string required

    Save data onChange. Default's to true if not appllied.

  • data-document_id string required

    Save data onChange. Default's to true if not appllied.

  • name string required

    Save data onChange. If attribute is empty or not used Default's to true.

  • data-read_value boolean optional

    Fetch value on page load or when updated. If attribute is empty or not used Default's to true.

  • data-save_value boolean optional

    Save data onChange/submitBtn. If attribute is empty or not used Default's to true.

  • contenteditable boolean optional

    htmltags can be editable

  • data-cursors boolean optional

    Save data onChange/submitBtn. If attribute is empty or not used Default's to true.

Forms

Using your forms with CoCreateJS Realtime CRUD is super easy, Form is used for grouping elements and applying attributes dynamically to it childeren. form tag can be left empty or you can apply attributes you would like to apply to childeren elements. attributtes will only be applied to childeren if the attribute does not already exist. If form realtime false it will require submitBtn

					<form data-realtime="true" data-collection="module_activity">

Inputs

Now that you have the form tag in place lets add some new inputs or update your current inputs connecting them to the CoCreateJS.

					<input type="text" data-Collection="module_activity" data-document_id="" name="name">

Checkbox

Now that you have the form tag in place lets add some new inputs or update your current inputs connecting them to the CoCreateJS.

CRUD Objects

Create Document Parameters

  • namespace string optional

    create namespace

  • rooms string optional

    creates room

  • broadcast boolean optional

    broadcast or only save to db

  • collection string required

    collection to save to

  • data object optional

    data to save

  • element string optional

    Save data onChange/submitBtn. If attribute is empty or not used Default's to true.

							
  CoCreate.createDocument({
    namespace:'',
    room:'',
    broadcast: true/false, (default=ture)
    broadcast_sender: true/false, (default=true) 

    collection: "test123",
    data:{
      name1:“hello”,
      name2:  “hello1”
    },
    element: “xxxx”,
    metaData: "xxxx"
  }),

    Save data onChange/submitBtn. If attribute is empty or not used Default's to true.

    element: "xxxx",
    metadata: "",
    operator: {
    fetch: {
      name: 'xxxx',
      value: 'xxxxx'
    },
    filters: [{
      name: 'field1',
      operator: "contain | range | eq | ne | lt | lte | gt | gte | in | nin",
      value: [v1, v2, ...]
    }, {
      name: "_id",
      opreator: "in",
      value: ["id1"]
    }],
    orders: [{
      name: 'field-x',
      type: 1 | -1
    }],
    search: {
      type: 'or | and',
      value: [value1, value2]
    },

    startIndex: 0 (integer),
    count: 0 (integer)
    },

    is_collection: true | false,
    //. case fetch document case
    created_ids : [id1, id2, ...],


    -------- additional response data -----------
    data: [] // array
  }

Reference

Passing attributes is how you will create a flow. For example you may have a customer profile that when you click on the edit icon a form opens for editing the customers information.

For passing attributes we utilize <a> tag. Using </a> tag you can open a modal, movable window/modal or new page in the same tab, new tab or new window. And pass an id to insert into that page's html to dynamically display data in realtime.
