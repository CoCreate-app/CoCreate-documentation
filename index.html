Introduction

CoCreate.js is a Realtime CRUD and state management API. Easily intergrates in any HTML template. Converts a regular html website into a dynamic Realtime application. If you know basic html you can easily implement and use the CoCreate.Js library. A CoCreate api key is required to open a realtime socket channel. We are excited to see what you build.

Getting Started

Go to CoCreate.app/register and retrieve your script which will include the necessary api and security key. And paste it at the end before the ending body tag of any page where you would like to fetch and save data in real time. 


  <script>
    var config = {
      apiKey: "test123456",
      securityKey: "test123456",
      organization_Id: "5d0e4e6147fb95a3b8719f12"
    
  </script>

Realtime Form

Introduction

Connecting your form to the CoCreate.Js Realtime Crud is super easy, just make sure your form tag has the following attributes.

<form data-real_time="true" data-collection="module" data-form_id="unique_id">

You will have the option to set realtime as tru e or false. If data-realtime set to false then the form will require a button to save class=”submitBtn”. And yes there is a method for redirecting using a btn and >a> tag which we will discuss at length further in this documentation.

We have 2 collections by default “modules” and “module_activity”. If the data-collection="" is left empty or if the attribute is not used it will default to “module_activity”.

If your logic requires a new collection just add it to the attribute data-collection="custom collection name" and it will be created if it does not exist.

Adding attribute data-form_id="a unique id” allows for a form creating a new record to be updateable in Realtime. It also differentiates the form from all other forms in the case that you may have multiple forms on the same page.

Inputs

Now that you have the form tag in place lets add some new inputs or update your current inputs connecting them to the Crud. But first a few things there are a few things that we should note. You can NOT create a module_activty without having a data-module_id. So you will first need to create a module then use that id to begin creating records for that module using module_activity

<input type="text" data-module_id=""  data-module_activity_id="" name="name" data-fetch_value="true" placeholder="Name">
Select2

For select2 we will be introducing a few new attributes. The attribute data-collection_fetch is used to return all documents from a collection. The attribute data-module_fetch is used only if the collection is module-activty. It allows you to filter results by module_id. The module_id can be separated by commas to show a few, one, or all if left empty.

data-template_id is used to define a template that will be used to display the search results.

data-selected_template_id is used to define the template to use to display the selected results

<select data-collection_fetch="modules_activity" data-module_fetch="" data-module_activity_id="5d0e7c649cf73d2f4b3f97d0" name="test2" data-template_id="dropdown2" data-selected_template_id="selectedTemplate2" class="select2_ele" multiple="multiple" style="width: 100%"></select>
        
        <div id="dropdown2" style="display: none">
            <h5 data-module_activity_id="" name="first_name"></h5>
            <span data-module_activity_id="" name="last_name"></span>
        </div>
        
        <div id="selectedTemplate2" style="display: none">
            <h5 data-module_activity_id="" name="first-name"></h5>
             <span data-module_activity_id="" name="last-name"></span>
        </div>
Array of Inputs

Used to clone a group of inputs. Common use cases are items in an invoice, multiple addresses, phone numbers, etc

Data-array_name used to create a name for the group of inputs and values.

Data-item_template_id used to define the template that contains the inputs you would like cloned

Passing Attributes

Introduction

Passing attributes is how you will create a flow. For example you may have a customer profile that when you click on the edit icon a form opens for editing the customers information.

For passing attribute we utilize <a> tag. Using <a> tag you can open a modal, movable window/modal or new page in the same tab, new tab or new window. And pass an id to insert into that page's html to dynamically display data in reatime. 

<a href="builder.html" data-pass_collection="modules" data-pass_module_id="" data-pass_prefix="" data-pass_to="me1234">page builder.html></a>

Window and Modals

Introduction

We have intergrated dhtlmx windows as a movable modal with a set of attributes to control it from the front end and pass values to it to display dynamic content

data-pass_to="me1234"

Regular Modal

You can use any framework modal. By adding the id and trigger based on your prefered frameworks modal documentation. Combined with our passing attributes feature you can open dynamic data in modal.

Realtime Datatables

Introduction

We have intergrated Datatables.net with CoCreate.js to create realtime datatables. Just define the collection and columns and cocreate.js will display your data in realtime. 

<table id="example1" class="table table-striped table-bordered" style="width:100%" data-template_id="exampleTemplate">
            <thead>
                <tr>
                    <th>Name</th>
                    <th>Username</th>
                    <th>Email</th>
                    <th>Phone</th>
                    <th>Id</th>
                </tr>
            </thead>
            <tbody>
                <tr class="template">
                    <td><a href="datatable_update.html" data-pass_module_activity_id="" data-pass_to="me1234" target="_blank"><p name="name"></p></a><p name="email"></p></td>
                    <td><p name="username"></p></td>
                    <td><p name="email"></p></td>
                    <td><p name="phone"></p></td>
                </tr>
            </tbody>
